Tier A

Agentic Workflows

Production-ready

An agent is a model given tools and a multi-step task it works through with checkpoints. Where it earns its place: variable, document-heavy, multi-step work — classify an inbound item, extract its fields, draft a response, route it — with a human gate on anything consequential.

Where it is over-sold: Anything a deterministic script or rule does more cheaply and reliably. Selling an agent where a scheduled job wins is precisely the hype we do not trade in.

Sellable as: Productised workflows (intake triage & extraction, invoice reconciliation, research/summarisation) with a governance and audit spine built in.

Revenue shape: P Project build + R Recurring retainer

Owners: Emmit (build & PoC) with Vincent (governance, audit, access)

Reusable spine: Classify → Extract → Route → Draft → Human Gate → Audit

Re-skinned from restaurant to claims, tickets, FOI, AP processing.

Tier B

Python Production Apps & Integration

Strong demand

Custom services and pipelines — extraction and reconciliation engines, API and MCP integrations, scheduled processing — built in Python and handed over.

The honest line: AI accelerates the build dramatically, but 'production' means ownership — architecture, deployment, security, monitoring, integration with legacy and SAP, and maintenance. That is a Solution Architect's job, not a chat window's. The gap between a working prototype and a production system someone is accountable for is precisely the billable engagement.

Sellable as: Bespoke build + integration, plus a support/monitoring retainer — the annuity.

Revenue shape: P Project build + R Recurring retainer

Owners: Emmit (architecture, SAP, integration) with Colin (Microsoft & automation)

Tier C

Microsoft Stack — Power Platform & M365

High demand

The most important tier, because it is where our M365 clients already live. Power Apps, Power Automate flows, SharePoint as a document and data backend, Dataverse when scale justifies it — with a governed AI layer (Azure OpenAI, AI Builder, Copilot Studio) on top.

Licensing line — our credibility: M365 E3 includes seeded Power Platform only — no premium connectors, no full Dataverse, no AI Builder, no Copilot. Real production apps require Power Apps Premium (~US$20/user/month). Knowing exactly where the paid line falls is the consultant's value, not a footnote.

Sellable as: Power Platform builds on existing M365/E3 estates, licensing-and-governance advisory, Copilot-readiness and oversharing remediation, plus a managed retainer.

Revenue shape: P Project build + R Recurring retainer

Owners: Colin (Microsoft everything) and Holger (SharePoint, M365, creative & delivery), with Vincent on governance and Emmit on architecture.
